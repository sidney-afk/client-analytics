### Architecture change — write path (started 2026-06-15)
Until now, writes flowed **Sheets-first → Supabase-mirror (downstream)**, so a Google Sheets
outage could block a save (it degraded gracefully after the read-failure guard, but still didn't
persist until Sheets recovered). We are **inverting every writer** to:

> **Supabase-primary (authoritative write + the echo the FE adopts) → Google Sheet best-effort
> mirror (`onError: continueRegularOutput`).**

So a Sheets outage can never stop a save. The Sheet is **kept as a backup / human-readable mirror**
(NOT retired). Reads were already on Supabase; after this lands, Supabase is the source of truth for
both reads and writes. Same `updated_at` is written to both stores so parity stays exact.
