### Architecture change — write path INVERTED (DONE 2026-06-15)
Writes used to flow **Sheets-first → Supabase-mirror (downstream)**, so a Google Sheets outage could
block a save (the 2026-06-14 brownout, exec 68798/68814, where a hung Sheets read made the phantom-row
guard reject a valid Kasper approval). **Every writer is now inverted** to:

> **Supabase-primary (authoritative write + the echo the FE adopts) + reads existing rows from
> Supabase → Google Sheet best-effort mirror (`onError: continueRegularOutput`, parallel, terminal).**

A Google Sheets outage can no longer stop a save. The Sheet is **kept as a backup / human-readable
mirror** (NOT retired). Supabase is now the source of truth for **both reads and writes**. The same
`updated_at` is written to both stores so parity stays exact, and the FE webhook contract is unchanged
(same `{ok, post}` partial-patch echo).
