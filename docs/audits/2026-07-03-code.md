# SyncView (/home/user/client-analytics) — Full Audit

## 1. App map & deployment

**Deployment:** GitHub Pages from `main` (repo `sidney-afk/client-analytics`), custom domain via `CNAME` = `syncview.synchrosocial.com`. Merging to `main` ships immediately (Pages caches ~10 min). `404.html` redirects clean paths (`/onboarding_form`, `/ai_onboarding_form`) back into the SPA. Repo is **public** (docs warn never to commit secrets).

**Structure:** the entire app is ONE file, `/home/user/client-analytics/index.html` — 2,475,319 bytes, 36,022 lines, inline `<script>` (~1.67 MB JS), no build step, no modules (~1,174 top-level globals per SAMPLES_REBUILD_STRATEGY.md). CDN deps: supabase-js@2 UMD, chart.js@4.4.0, xlsx@0.18.5 (jsDelivr). A tiny pre-paint "boot gate" script in `<head>` (lines 25–105) tags `<html>` with boot mode (onboarding / intake / client / password / predicted tab) before first paint.

## 3. WRITE PATHS (every UI-triggered write; all POST JSON unless noted)

### Calendar → n8n `calendar-upsert-post` (const at 13156; payload `{client, post, comments_base_at}`; response `{ok, post(echo), conflict?}`) — 6 FE call sites + reconciler:
1. **Card edit save** `_calFlushCardSave` (fetch ~20423): any field edit (name, date, caption, links, per-component status, comments, approvals). Debounced ~650 ms per card; **field-level patch under v2** (only changed keys + always the four `*_tweaks` comment cells serialized); `comments_base_at:''` under v2 (skips scalar-conflict guard; server 3-way comment merge still runs via `calendar_merge_comments` RPC); `__CLEAR_LINK__` sentinel for intentional link clears. UX: fully optimistic, per-card "Saving…" chip, echo merged back, failed-save retry (`_calRetrySave`), realtime self-echo suppressed via `_calLastLocalWriteAt`.
2. **Settings row** `_calSaveSettings` (14216): kebab toggles (collab mode, YouTube-title review, enabled_platforms) written as pseudo-post `id=CAL_SETTINGS_PID name='__cal_settings__'` with settings JSON in `caption`; serialized write chain, 3 retries w/ backoff, optimistic + localStorage-durable.
3. **Bulk import** `_calBulkUpsertPosts` (16365): whole-post sequential sends, 350 ms gaps, verify-and-retry passes (n8n acks before Sheets commit — historical race).
4. **Archive** `_calArchiveOne` (20828): `{id, status:'Archived'}`; bulk archive pooled at 6 concurrent (`_calRunPooled`); optimistic removal; waits out in-flight saves first.
5. **Linear-tab import** (~25109): after video/graphic form submit, FE writes cards directly (deterministic `p_lin_<ident>` ids), 200 ms gaps, durable background job.
6. **Kasper persist** `_kasperPersistPostWrite` (34208): approvals/tweak requests/comments/finish/close; field-level patch vs `_patchBase` snapshot, per-card serialized chain; sends `comments_base_at` = real `_baseAt`; then pushes each sub-status to Linear.
- `calendar-append-post` and `calendar-delete-post` consts (13157–58) are **defined but never called** — dead endpoints (creates & archives go through upsert).

### Calendar reorder: `calendar-reorder-batch` (primary, 21967) → `calendar-reorder` (fallback, 21974). `{client, items:[{id, order_index}]}`. Serialized + coalesced; optimistic pin for 12 s (`_calReorderOptimistic`); success = "New order saved" toast with **Undo**; failure = revert + toast.

### Calendar → Linear (shared with SXR):
- `linear-set-status` `{issue:<sub-issue url>, status}` (15392, 28573): fired on every component status change with a linked issue. Per-issue serialized + coalesced (newer status supersedes queued older); failure → **durable localStorage outbox** retried on focus/interval. "Posted" never pushed; samples also never push Scheduled.
- `linear-add-comment` `{issue, body, author}` (15442, 28582): client/Kasper tweak text mirrored to the editor's Linear inbox; fire-and-forget + outbox.
- `send-urgent-slack` `{issue, client, name}` (15491): URGENT button (SMM calendar + Kasper card); confirm dialog → button "Sending…" → latches "Sent" for the session; n8n resolves editor from Linear + Video Editors sheet, posts to #video-editing.

### Caption AI: `generate-caption` `{client, postId, assetUrl, captionPrompt, jobId}` (21276) — job rows in n8n data table `caption_jobs`; UI polls `caption-job-status`, cancels via `caption-job-update` `{jobId, cancel_requested:true}`; progress chip scraping→transcribing→writing; survives refresh. `caption-prompts-save` (21697) saves per-client prompt.

### Samples Old: `samples-upsert` `{client, sample:<whole wire row>}` (11822; debounce 700 ms; chip "Saving…"/"Saved"/**"Saved on device"** — failure silently stays local-only); delete = upsert `{id, status:'Archived'}` fire-and-forget (11766); `samples-reorder` `{client, items}` (11797, errors swallowed).

### Samples New (SXR): `sample-review-upsert` `{client, sample: patch|whole, comments_base_at:''}` — `_sxrFlushCardSave` (27445, calendar-parity optimistic funnel, field-level patch, `__CLEAR_LINK__`, thumb_rev bump), `_sxrArchiveOne` (27049, optimistic + rollback on failure), `_sxrKasperPersist` (29057, per-component approve/comment/approve-after-tweaks/request-change, then Linear push). `sample-review-reorder` (27263, serialized/coalesced, rollback + notify on failure). - Card columns `linear_issue_id` (VID sub-issue URL) + `graphic_linear_issue_id` (GRA) on both `calendar_posts` and `sample_reviews`; parent-issue links are detected and refused (~15520); bulk-link modal matches "Video N" sub-issues via `linear-subissues`.
- **Inbound** Linear→card: Linear org webhook → n8n `linear-status-sync` (id `MJbMZ789B5ExZz9x`; samples branch "Handle Sample Linear Event" inside the same workflow) → GraphQL fetch → `mapStatus` → minimal `{id, video_status|graphic_status}` patch POSTed through `calendar-upsert-post`/`sample-review-upsert`. - **Outbound** card→Linear: `linear-set-status` (team-aware, silently skips absent states) + `linear-add-comment`, with per-issue serialization + durable outboxes; point-adoption of Linear status when a link is freshly set; stale-regress re-assert (`_calReassertLinearStatus`, 20 s throttle).
- Status map `_calMapLinearStatusStrict` (tweak/scheduled/posted/approved/smm/kasper/client/backlog…); `CAL_PRIORITY` exists in **three lock-step copies** (FE, n8n handler, reconciler — which extracts the FE functions from index.html at runtime via `grabFunc`/`new Function`).
- **Reconcilers** (the guarantee): `scripts/linear-sync-reconcile.js` + `scripts/sample-linear-reconcile.js`, GitHub Actions `linear-sync-reconcile.yml` / `sample-linear-reconcile.yml` cron `*/10 * * * *` (real ~10-min cadence driven by n8n trigger workflows `AkiFmromoDkmsh39` / `ZJOtYpQZj73DcBB1` firing workflow_dispatch; GitHub cron alone averages ~3.4 h). Most-recent-action-wins using DB-stamped `video_status_at`/`graphic_status_at` (Postgres BEFORE trigger), SAFETY_CAP=15 aborts on mass divergence, ledger in Actions cache, no secrets (public webhooks + anon key).

## 6. Auth / security model
- Staff gate: hardcoded client-side password `synchrosocial2026` (`submitPassword`, index.html:25721) → `localStorage.syncview_auth_v1='ok'`. Purely cosmetic — no server enforcement.
- Kasper (`?Kasper=1`→sessionStorage `syncview_kasper_unlocked`) and TikTok-pilot (`?ttpilot=1`) are URL-param unlocks, also cosmetic.
- Client links: `?c=<clientName>&t=<token>`; token compared against Clients Info sheet column `client_review_token`; **fails OPEN with a console warning when the column is empty**. Read-only/role gating (`_isClientLink`, comment roles smm/client/kasper) is all front-end.
- Browser ships the committed publishable anon key; RLS = anon SELECT `using(true)` on operational tables (any client link can read EVERY client's rows — flagged in AUDIT_2026-06-15), anon INSERT→401. All n8n write webhooks are unauthenticated with CORS `*` — the write path's only "auth" is obscurity. Exceptions: credentials tables (no anon access, passphrase-gated Edge Function) and onboarding_fallback (service-role only).
- Known outstanding: rotate plaintext Linear API key in n8n Code nodes; rotate n8n service_role credential `XdBpJ6Xk8PMpZXXT`; per-client RLS deferred until real auth.
