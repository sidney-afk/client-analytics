## UNKNOWNS (could not verify, and why)

1. **pg_policies / live grants**: not queryable via anon REST (no SQL, no pg_catalog exposure).
   Posture inferred from behavior (200 vs 401) + the 07-03 owner dump. The legacy tables'
   anon INSERT/UPDATE/DELETE grants (RLS-only barrier) can't be re-confirmed read-only.
2. **Realtime publication membership** for the 4 new tables: `pg_publication_tables` is privileged.
   Migrations contain guarded ADDs and the tables exist + flags seeded, so application is likely
   complete (INFERENCE), but membership needs owner confirmation (one SQL-editor query).
3. **Edge Function list / versions / secrets / verify_jwt state**: management-API only. Strong
   indirect evidence of deployment (templates & caption_prompts rows `updated_by='SyncView'` on
   07-04; gate doc records `templates-save`/`caption-prompts-save` active v1). I deliberately did
   not GET/OPTIONS any `functions/v1/*` endpoint — that executes code, violating the read-only rule.
4. **PITR / plan tier / backup state** (spec §7): invisible to anon REST; docs imply free tier where
   PITR is unavailable → **owner confirmation required** (flagged in implications #6).
5. **Anon write rejection**: not re-tested (writing is forbidden by this audit). Carried from
   baseline documentation of 401-on-INSERT.
6. **Column types beyond JSON serialization**: REST shows values, not pg types; typing carried from
   the 07-03 baseline dump (no shape contradictions observed).
7. **Cause of sample_review_events id gaps (~37) and calendar_post_events ids 1–33**: deletions vs
   sequence-consuming failed inserts is indistinguishable from REST; either way rows are absent now.
8. **Whether the samples Linear branch would write ledger events if a real client used SXR**: zero
   non-UI events all-time is compatible with "code never inserts" and with "path never ran"; needs
   the n8n-domain auditor to inspect the live `Handle Sample Linear Event` branch for an insert node.
9. **RPC presence/signatures** (`calendar_merge_comments`, `sample_review_merge_comments`): the
   OpenAPI root is empty and calling an RPC would execute it; existence carried from the 07-03 dump.
